Introduction

Thoughtmarks is a cognitive capture tool designed for people who think faster than they can organize. We're not another note-taking app. We're the app that catches ideas before they vanish.

This brand guide is based on the actual deployment at thoughtmarksapp.com and establishes the visual and verbal standards that ensure Thoughtmarks remains recognizable, consistent, and true to its core identity across all touchpoints.

Brand Essence

What We Are: A voice-first cognitive capture tool with AI organization

What We're Not: Another note-taking app, voice memo app, or productivity system

Brand Personality: Smart but not pretentious, fast and efficient, supportive of chaos

Core Positioning

"The app for people who think faster than they can organize. Capture ideas at the speed of life. AI organizes them automatically. Find anything with semantic search. Your brain can finally stop trying to remember everything."

Key Changes in v3.0

  • Primary Accent: Blue (#3B82F6) is the primary interactive color, NOT yellow
  • Brand Yellow: Reserved for tagline script text ONLY (#FEFF00)
  • Backgrounds: Warm dark (#09090B) not pure black
  • Typography: Lato (primary), Over the Rainbow (tagline only)
  • Design Effects: Subtle gradients and blur effects ARE used

03

Color Usage

Primary Blue (#3B82F6)

This is THE primary interactive color. Use for all buttons, CTAs, links, and interactive elements.

  • DO: Use for primary buttons and CTAs
  • DO: Use for links and focus rings
  • DO: Use for glow effects and accents
  • DON'T: Use yellow for buttons (yellow is tagline-only)
  • DON'T: Use other accent colors (blue is primary)

Brand Yellow (#FEFF00)

This is ONLY for tagline script text. NEVER use for buttons or primary CTAs.

  • DO: Use for tagline "bookmarks for your brain" script text
  • DO: Use in iOS icon bookmark element
  • DON'T: Use for buttons or primary CTAs
  • DON'T: Use for large background areas (causes eye strain)
  • DON'T: Use for body text (readability issues)

Warm Dark Backgrounds

Dark mode is our default. Use warm dark backgrounds, not pure black.

  • Primary background: Base (#09090B) - warm dark, not pure black
  • Card surfaces: Elevated (#18181B) - for cards and elevated surfaces
  • Borders/hover: Surface (#27272A) - for borders and hover states

Accessibility Notes

  • Blue on white: AAA contrast
  • Blue on warm dark: AA contrast
  • Yellow on black: AAA contrast (21:1 ratio)
  • Always test color combinations for accessibility

Typography Hierarchy

Type Scale

Element Font Weight Size Case
Wordmark Lato Black (900) Variable ALL CAPS
Tagline Over the Rainbow Regular Variable lowercase
H1 Headlines Lato Black (900) 3rem (48px) Sentence case
H2 Subheads Lato Bold (700) 2rem (32px) Sentence case
H3 Lato Semibold (600) 1.5rem (24px) Sentence case
Body Text Lato Regular (400) 1.125rem (18px) Sentence case
Small Text Lato Regular (400) 0.875rem (14px) Sentence case

Typography Rules

  • All caps for THOUGHTMARKS wordmark only
  • Sentence case for most headlines (more approachable)
  • Generous white space - let the copy breathe
  • Never center-align body text
  • Use Lato for everything except tagline
  • Tagline always uses Over the Rainbow script in yellow

Do's and Don'ts

Visual Do's

DO
  • Use blue (#3B82F6) for all buttons and CTAs
  • Use yellow (#FEFF00) ONLY for tagline script text
  • Use warm dark backgrounds (#09090B, #18181B)
  • Use Lato for all UI text
  • Use subtle gradients with blur effects
  • Maintain generous white space
DON'T
  • Don't use yellow for buttons or primary CTAs
  • Don't use pure black backgrounds
  • Don't use yellow for anything except tagline
  • Don't use script font for anything except tagline
  • Don't avoid gradients (subtle ones ARE used)
  • Don't clutter designs with too many colors

Messaging Do's

DO
  • Use active verbs: capture, build, remember
  • Acknowledge the chaos
  • Be specific about benefits
  • Short, punchy, scannable sentences
  • Use brain/neural metaphors sparingly
DON'T
  • Don't use productivity jargon
  • Don't promise unrealistic outcomes
  • Don't talk down to users
  • Don't overcomplicate the value prop
  • Don't use emojis in brand materials
